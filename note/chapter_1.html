🏭 流水线技术 公式必背
洗车流水线：第1辆烘干时，第2辆擦洗，第3辆打泡，第4辆冲水——每环节不闲着
⏱️ 流水线执行时间（各段时间不等）： T = (t₁ + t₂ + ... + tₖ) + (n-1) × Δt 其中：k = 段数 | n = 指令数 | Δt = 最长段时间 ⚡ 加速比（各段时间相等时）： S = (n × k × Δt) / [k×Δt + (n-1)×Δt] 当 n→∞ 时，S → k（最大加速比 = 流水线段数） 📊 吞吐率： TP = n / T 最大吞吐率 = 1/Δt
💡 例题：3段流水线，各段耗时 2ns/3ns/1ns，100条指令
T = (2+3+1) + (100-1)×3 = 6 + 297 = 303 ns
流水线冒险原因说明
结构冒险资源冲突多条指令竞争同一硬件资源
数据冒险数据依赖后指令需要前指令结果
控制冒险分支跳转条件分支导致流水线清空

Cache 命中率公式 必背公式
命中率 H = Cache命中次数 / 总访问次数 方式① 同时访问： t_avg = H × t_c + (1-H) × t_m 方式② 先Cache再主存（未命中后再访问主存）： t_avg = H × t_c + (1-H) × (t_c + t_m) 例题：t_c=10ns, t_m=200ns, H=95% 方式① = 0.95×10 + 0.05×200 = 9.5+10 = 19.5ns 方式② = 0.95×10 + 0.05×(10+200) = 9.5+10.5 = 20ns ←常考

🛤️2.6 总线系统
总线类型功能说明
数据总线运输数据位数决定一次传多少（32位=4字节/次）
地址总线门牌号系统n位地址总线 → 最大寻址 2ⁿ 字节（32位→4GB）
控制总线交通信号灯控制读写、中断等信号
总线带宽 = 总线宽度 × 总线频率 (MB/s) 例：64位总线，800MHz → 带宽 = 64/8 × 800 = 6400 MB/s
带宽计算：为什么除以 8？ 避坑指南
💡 核心原因：单位换算！
总线宽度的单位是位（Bit），表示一次能并行传多少个 0 或 1。
而带宽的统计标准单位通常是字节（Byte）（如 MB/s）。
因为 1 Byte = 8 Bits，所以在计算带宽时，必须将总宽度除以 8 来完成单位转换。
串行总线 vs 并行总线
🚗 生活类比：单车道高铁 vs 多车道公路
并行总线 = 多车道公路：多辆车并排开。距离长了容易互相干扰，且终点必须等最慢的车对齐，所以上限开不快。
串行总线 = 单车道高铁：每次只走一节车厢。因为没有旁边车道干扰，无需对齐，发车频率（时钟频率）可以提升到极其恐怖的程度。
结论：现代总线中，串行总线靠“极高的频率”在总速度上彻底碾压了并行总线！
并行总线 (Parallel)
传输方式多根线同时传多位
时钟频率很难提高(防干扰)
传输距离仅限短距离
现代代表传统 PCI、PATA
串行总线 (Serial)
传输方式一根线逐位(bit)传
时钟频率极高(无对齐负担)
传输距离适合长距离
现代代表PCIe、SATA、USB

🏨3.2 操作系统 — 进程管理 高频考点
操作系统 = 酒店总管家：进程管理=分配客房 | 存储管理=管理仓库 | 文件管理=档案室
操作系统的四大特征
🏠 生活类比：合租房日常
并发 = 室友A在洗衣服，室友B在做饭（微观上可能在切换，宏观上都在干活）。
共享 = 所有人都可以共同使用客厅和冰箱。
虚拟 = 房东把一个大客厅打隔断变成了好几个房间，每个人都觉得自己有个独立卧室。
不确定性 = 每天谁先抢到卫生间、谁先出门，顺序和耗时全凭运气，无法预知。
并发性 (最基本)
宏观上同时运行，微观上交替执行。
注：区别于“并行”（同一物理时刻发生）
共享性
系统资源供内存中多个并发执行的进程共同使用。
虚拟性
通过技术（如分时、分页）将一个物理实体映射为多个逻辑对应物。
不确定性(异步)
进程以不可预知的速度向前推进，每次执行结果的发生时间不确定。
操作系统的分类
分类类型一句话要点生活化类比
批处理系统无交互，批量处理，吞吐量大攒衣服洗：攒够一堆脏衣服，扔进洗衣机一次性全洗完。
分时系统时间片轮转，多终端交互性强餐厅服务员：服务员给每桌轮流倒水，每桌都觉得有专属服务。
实时系统严格的时间限制，极高的可靠性汽车安全气囊：撞车瞬间必须立刻弹出，晚半秒都不行。
网络系统通过网络实现资源共享和通信办公室共享：多台独立电脑通过局域网，共用一台打印机。
分布式系统多机协同计算，对用户完全透明跨国集团：全球各地员工协同做一个大项目，客户只对接公司主体。
嵌入式系统专用于特定设备，软硬件可裁剪智能电饭煲：系统烧录在芯片里，专干精确控温煮饭这一件事。
进程三态模型
进程调度算法
算法说明特点
FCFS先来先服务公平但对短作业不利
SJF短作业优先平均等待时间最短，可能饿死长作业
RR时间片轮转公平，适合分时系统
优先级调度按优先级执行可能饿死低优先级进程
PV操作（信号量）常考
P操作（Wait）
信号量-1，若<0则进程阻塞等待
V操作（Signal）
信号量+1，若≤0则唤醒一个等待进程
死锁四个必要条件
#条件含义
1互斥条件资源同一时间只能被一个进程占用
2请求与保持占着资源还要申请新资源
3不可剥夺不能强制取走进程已占资源（注意是"不可"剥夺！）

⚠️ 考题"可剥夺条件"——注意正确说法是「不可剥夺」，"可剥夺"不是死锁条件！
概念说明
进程资源分配的基本单位，有独立地址空间

🧩3.7 构件 vs 中间件 易混辨析
🚗 生活类比：汽车零件 vs 物流网络
构件 (Component) = 标准化零件（如米其林轮胎）：隐藏内部细节，提供标准接口，主要为了复用和解耦。它是造应用的“砖块”。
中间件 (Middleware) = 跨国物流网络（如顺丰快递）：屏蔽底层复杂路线和海关（操作系统/底层网络），负责把包裹（数据）送达。它是连接应用的“公路”。
核心定位对比
构件 (Component)
层级位置应用层（业务逻辑砖块）
核心目标代码复用、降低耦合
颗粒度较小（类、模块、微服务）
关注点业务功能怎么实现
中间件 (Middleware)
层级位置基础层（OS与应用间）
核心目标通信、集成、高可用
颗粒度庞大（独立部署的集群）
关注点应用间数据怎么搬运
常见分类与标准 必考映射
类别特点与说明典型代表 / 标准
构件标准跨语言、跨平台，最中立CORBA (OMG组织)
Windows平台为主COM/DCOM (Microsoft)
基于Java平台EJB (Sun/Oracle)
中间件分类统一访问不同数据库数据库中间件 (JDBC/ODBC)
异步消息传递、削峰填谷消息中间件/MOM (RabbitMQ/Kafka)
管理分布式事务交易中间件/TPM (Tuxedo)
调用远程服务RPC中间件 (gRPC/Dubbo)
🧠 协同关系：它们是天作之合
构件往往运行在中间件之上。
例如：你用 Java 写了一个业务构件（EJB），你把它部署在提供运行环境和通信支持的应用服务器（对象中间件）上。两者配合，才能支撑起大型系统。
🧠 速记口诀
中间件分类：「数消交对远」
构件标准：「CORBA万国通，COM微软用，EJB爪哇种」

代语言类比特点
第1代机器语言原始人手势纯0和1，CPU直接执行
第2代汇编语言象形文字助记符代替二进制，需汇编器
第3代高级语言现代语言接近人类语言（C/C++/Java/Python），需编译/解释
第4代建模语言 (UML)工程蓝图可视化建模，面向对象，所见即所得，如 UML
UML 核心三要素：事物、关系、视图
1. UML 事物 (Things) —— 模型的“词汇”
结构事物 (名词)
模型的静态部分。如：类(Class)、接口(Interface)、用例(UseCase)、构件(Component)、节点(Node)。
行为事物 (动词)
模型的动态部分。表示跨越时间和空间的行为。如：交互(Interaction)、状态机(State Machine)。
分组事物 (整理箱)
模型的组织部分。最主要的就是：包(Package)，用来把元素组织成组。
注释事物 (便签纸)
模型的解释说明部分。如：注解(Note)，用于添加文本注释。
2. UML 关系 (Relationships) —— 词汇的“连接”
关系类型图形表示含义与类比
依赖 (Dependency)带箭头的虚线“使用”关系：一个事物的变化会影响另一个事物。（如：人依赖空气、司机依赖车）
关联 (Association)带箭头或不带的实线“拥有”关系：结构化关系，一种强依赖。（如：企鹅与气候）
特例：聚合(空心菱形，整体与部分可分)、组合(实心菱形，同生共死)
泛化 (Generalization)带空心三角箭头的实线“继承”关系：一般与特殊的关系。（如：动物与狗，狗泛化自动物）
实现 (Realization)带空心三角箭头的虚线“接口与实现”：类实现了接口的功能。（如：键盘实现了输入设备接口）
🧠 关系速记口诀
「泛化实，实现虚；关联实，依赖虚」
记忆法：三角箭头代表“族谱/继承”（实线是亲生继承，虚线是契约实现）；普通箭头代表“使用”（实线是长期拥有的关联，虚线是临时借用的依赖）。
3. UML "4+1" 视图模型 必考选择题
🏗️ 架构师视角：一栋大楼的五张图纸
给客户看功能（用例）、给设计师看空间（逻辑）、给包工头看施工流（进程）、给泥瓦匠看砖块怎么砌（实现）、给物业看大楼水网电网接哪（部署）。
视图名称主要关注者关注点与作用对应 UML 图
用例视图 (Use Case)最终用户 / 分析师系统的功能需求。它是其他所有视图的核心和驱动力（所以叫"1"）。用例图
逻辑视图 (Logical)系统分析 / 设计人员系统内部的功能设计和结构（词汇表）。类图、对象图
进程视图 (Process)系统集成人员系统的非功能性需求（并发、同步、性能、吞吐量）。活动图、状态机图
实现视图 (Impl)程序员软件代码的静态组织结构、软件模块划分。构件图(组件图)
部署视图 (Deployment)系统/网络工程师软件到物理硬件的映射，拓扑结构、安装和通信。节点图(部署图)

🎵5.2 音视频数据量计算 必考公式
音频数据量(bit) = 采样频率(Hz) × 量化位数(bit) × 声道数 × 时间(s) 例：CD音质 = 44100 × 16 × 2 × 60 ÷ 8 = 84,672,000 字节 ≈ 10.09 MB/分钟 图像数据量(bit) = 水平像素 × 垂直像素 × 颜色深度(bit) 例：1920×1080，24位真彩色 = 1920 × 1080 × 24 ÷ 8 = 6,220,800 字节 ≈ 5.93 MB
💡 奈奎斯特采样定理：采样频率 ≥ 2 × 最高信号频率
人耳20Hz~20kHz → CD采样率44.1kHz > 2×20kHz
颜色深度色彩数

16位65,536色
24位1677万色（真彩色）
