SECTION 03

信息加解密技术

对称加密 · 非对称加密 · 算法参数 · 考点密度 ★★★★

🔑3.1 对称 vs 非对称 — 核心对比 ★★必考
对称加密（共享密钥算法）
密钥关系加密=解密（同一密钥）
速度快
主要用途大量数据加密
缺点密钥分发困难
代表算法DES / 3DES / IDEA / AES / SM4
类比夫妻共用一把家门钥匙
非对称加密（公钥加密算法）
密钥关系加密≠解密（公钥+私钥）
速度慢（比对称慢约1000倍）
主要用途密钥交换、数字签名
优点无密钥分发问题
代表算法RSA / SM2 / ECC
类比邮箱：公开投递口 + 私人钥匙
💡 非对称加密比对称加密慢约 1000 倍，所以大量数据加密用对称，密钥交换用非对称
🛡️3.2 对称加密算法参数 高频考点
类比：不同保险箱规格 — 分组长度=保险箱大小，密钥长度=钥匙复杂度

🧠 超强记忆口诀
「DES三重变112，IDEA 128全球通，AES替代脆DES，SM4国密双128」
算法分组长度密钥长度关键特点
DES64位56位最基础，已脆弱
3DES64位112位（2×56）DES改进版
IDEA64位128位全球通用标准
AES128位128/192/256位替代DES，可软硬件实现
SM4128位128位国密对称算法
⚠️ 3DES密钥长度是112位（2把56位），不是168位！
⚠️ AES支持128/192/256位密钥，不支持512位！
📮3.3 非对称加密算法 ★★必考
类比：邮箱系统
公钥 = 邮箱投递口（谁都能往里投信）· 私钥 = 开邮箱的钥匙（只有你有）
🔐 保密通信
加密者用接收方的公钥加密
解密者用接收方的私钥解密
效果只有接收方能看
类比用对方邮箱投信
✍️ 数字签名
签名者用发送方的私钥签名
验证者用发送方的公钥验证
效果证明是我发的
类比在合同上按手印
🧠 速记口诀
「公加私解为保密，私加公解为签名」
保密：用接收方的公钥 · 签名：用发送方的私钥
两大非对称算法
算法安全性基础特点
RSA大素数分解困难性国际通用，密钥2048位，比对称慢1000倍
SM2椭圆曲线离散对数国密算法，同等安全下密钥比RSA更短
⚠️ 保密通信中，甲发给乙，甲用乙的公钥加密，乙用乙自己的私钥解密！

🚪4.2 访问控制 ★考点
类比：小区门禁系统
主体 = 来访者 · 客体 = 要进入的场所 · 控制策略 = 门禁规则 · 审计 = 监控录像（防止门卫滥用权力）
🧠 速记口诀
三要素：「主客策」 → 主体 + 客体 + 控制策略
三内容：「认策审」 → 认证 + 控制策略实现 + 审计（防止滥用权力）
4种访问控制实现技术 易混淆考点
技术存储方式特点
访问控制矩阵（ACM）矩阵（行=主体，列=客体）基础形式，元素多时存储困难
访问控制表（ACL）按列（客体）保存最流行、使用最多！
能力表（Capabilities）按行（主体）保存—
授权关系表只存非空元素适合稀疏矩阵，用于安全数据库
🧠 速记口诀
「ACL按列客，能力按行主，授权取非空，矩阵是基础」
ACL = Access Control List（列表 = 按列存）
⚠️ ACL按客体（列）存储，能力表按主体（行）存储 — 别搞反！
⚠️ ACL是最流行、使用最多的访问控制实现技术！
✍️4.3 数字签名 ★★高频
类比：在合同上按手印
证明这份文件确实是你发的（不可抵赖）· 别人不能伪造（不可伪造）· 合同不能被改（不可改变）
数字签名5个必须满足的条件
可信

不可伪造
造
不可重用
用
不可改变

不可抵赖

🧠 速记口诀
「信造用变赖」 → "相信造出来用着变赖了" 😂
数字签名 = 公钥加密技术 + 数字摘要技术
①
对文件做摘要
用哈希函数（MD5/SHA）对原文生成固定长度摘要
先把文件压缩成指纹
②
私钥签名摘要
用发送方的私钥对摘要加密（签名）
用私章盖在指纹上
③
公钥验证签名
接收方用发送方的公钥解密，重新计算摘要比对
用公开信息验证私章
💡 先对文件做摘要再签名，是为了提升速度（摘要比原文小得多）
⚠️ 数字签名用到两种技术：公钥加密技术 + 数字摘要技术！
SECTION 05

信息安全的抗攻击技术

DoS/DDoS · 欺骗攻击 · TCP/IP攻击 · 考点密度 ★★★★

💀5.1 拒绝服务（DoS / DDoS）攻击 高频考点
类比：往餐厅塞满假客人
真客人进不来（合法用户无法访问）· 餐厅资源被耗尽（可用性被破坏）
DDoS（分布式） = 从四面八方同时涌来的假客人大军
要素内容
破坏的安全要素可用性（最常见考点！）
DDoS三级结构Client（客户端）→ Handler（主控端）→ Agent（代理端）
最广泛DoS方式SYN Flooding（同步包风暴）
🧠 速记口诀
DDoS三级结构：「客主代」 → 客户端 + 主控端 + 代理端
传统DoS4种模式：「耗破物错」 → 耗资源 + 破坏配置 + 物理破坏 + 利用错误
DoS防御4方法：「特防统修」 → 特征识别 + 防火墙 + 统计流量 + 修正漏洞
🎭5.2 欺骗攻击 — 三大对比 ★考点
攻击类型攻击原理防范方法
ARP欺骗 解析 IP→MAC 地址机制被欺骗 固化ARP表 · ARP服务器 · 双向绑定 · 防护软件
DNS欺骗 解析域名→IP 机制被欺骗 被动监听检测 · 虚假报文探测 · 交叉检查查询
IP欺骗 修改IP报头，伪装来源IP 防火墙
🧠 速记口诀
「ARP骗MAC，DNS骗域名，IP改报头」
DNS欺骗3种检测：「被虚叉」 → 被动监听 + 虚假报文探测 + 交叉检查查询
💻5.3 针对TCP/IP堆栈的攻击
攻击类型原理防范措施
SYN Flooding
（同步包风暴）		 攻击TCP三次握手，耗尽连接资源 减少等待超时时间
Ping of Death
（ICMP攻击）		 攻击操作系统网络层缓冲区 打补丁、升级系统
SNMP攻击 SNMP V1缺少认证机制 升级到V2以上，设置访问密码
⚠️ SYN Flooding 是应用最广泛的DoS攻击方式！
⚠️ Ping of Death 属于 ICMP攻击，不是SYN攻击！

🌐6.2 安全协议对比 ★★高频
